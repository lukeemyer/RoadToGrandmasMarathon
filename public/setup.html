Marathon Training Tracker Setup Wizard

Set up your own
training tracker

Follow these steps to deploy your own copy connected to your Strava account. Takes about 15 minutes.

1
Fork the repository Done

Fork the GitHub repo to get your own copy you can customize and deploy.

  1. Open the repository on GitHub and click Fork in the top-right corner.
  2. Choose your personal account as the owner. Leave the repo name as-is or rename it.
  3. Click Create fork.
Fork on GitHub →
2
Deploy to Vercel Done

Create a free Vercel account (if needed) and import your fork as a new project.

  1. Go to vercel.com/new and sign in with GitHub.
  2. Click Import next to your forked repo.
  3. Leave all settings as defaults and click Deploy. The first deploy will fail — that's expected, you haven't set env vars yet.
  4. Note your Vercel project URL (e.g. your-project.vercel.app).
3
Add Upstash Redis database Done

The tracker stores your runs in a small Redis database. Vercel's Upstash integration makes this free and automatic.

  1. In your Vercel project, go to the Storage tab.
  2. Click Create Database → Upstash Redis.
  3. Choose a region close to you and click Create & Continue.
  4. Click Connect to link the database to your project. Vercel auto-adds KV_REST_API_URL and KV_REST_API_TOKEN as env vars.
4
Create a Strava API app Done

You need a Strava API application to read your activity data. This is free and takes 2 minutes.

  1. Go to strava.com/settings/api.
  2. Fill in the application form. The fields don't matter much — use any name (e.g. "My Tracker"), category "Other", and website URL.
  3. Set Authorization Callback Domain to your Vercel domain (e.g. your-project.vercel.app). No https://, no trailing slash.
  4. Click Create. You'll see a Client ID and Client Secret — keep this tab open.
5
Set environment variables Done

In your Vercel project, go to Settings → Environment Variables and add these:

STRAVA_CLIENT_ID Your Strava app's numeric Client ID
STRAVA_CLIENT_SECRET Your Strava app's Client Secret
STRAVA_VERIFY_TOKEN Any random string (e.g. mysecrettoken123). Used to verify webhook delivery.
SHARED_SECRET A password for admin access to your tracker. Choose something strong.

The KV_REST_API_* variables were added automatically when you connected Upstash in step 3.

After adding all variables, redeploy the project:

  1. In Vercel, go to the Deployments tab.
  2. Click the three-dot menu on the latest deployment and select Redeploy.
  3. Wait for the deployment to show Ready.

Once deployed, click the button below to verify the environment is configured correctly:

6
Connect your Strava account Done

Authorize your tracker to read your Strava activities. This stores an access token in your Redis database.

Click the button below — you'll be redirected to Strava, asked to authorize, then brought back here.

7
Register Strava webhook Done

The webhook lets Strava push new activities to your tracker automatically — no manual syncing needed after each run.

Enter your SHARED_SECRET (the admin password you set in step 5), then click Register.
